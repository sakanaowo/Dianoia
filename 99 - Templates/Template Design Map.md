# Template Design Map

## Nguyên tắc chung

Mỗi template có thể có đủ 3 lớp, nhưng khi viết note thật không bắt buộc điền hết:

1. **Identity** — note này là gì?
2. **Context** — note này thuộc domain/theme/source/project nào?
3. **Extraction** — từ note này cần tách ra concept, atomic note, argument, question nào?

## Vai trò từng loại note

| Loại note | Vai trò chính | Câu hỏi cần trả lời |
|---|---|---|
| Domain MOC | Bản đồ một lĩnh vực | Lĩnh vực này gồm những khái niệm, câu hỏi, tranh luận nào? |
| Theme MOC | Bản đồ một theme xuyên lĩnh vực | Theme này nối các domain nào lại với nhau? |
| Concept Note | Làm rõ một khái niệm | Khái niệm này là gì, dùng để hiểu điều gì? |
| Atomic Note | Giữ một insight nhỏ | Mệnh đề nào đáng tái sử dụng? |
| Argument Note | Phân tích lập luận | Claim, evidence, assumption, objection là gì? |
| Comparison Note | Phân biệt hai thứ dễ nhầm | Khác nhau ở điểm nào, dùng khi nào? |
| Question Note | Giữ câu hỏi mở | Cần biết gì để trả lời tốt hơn? |
| Source Note | Giữ nguyên liệu từ nguồn | Nguồn nói gì, claim nào cần verify, extract note nào? |
| Book/Paper Note | Xử lý nguồn dài / học thuật | Thesis, method, contribution, objection là gì? |
| Person Note | Bản đồ một người | Người này giải quyết vấn đề gì, đóng góp gì? |
| Project Note | Điều phối hành động | Goal, scope, next actions, risks là gì? |
| Problem Note | Làm rõ vấn đề | Problem, constraints, unknowns, trade-offs là gì? |
| Daily/Weekly Review | Bảo trì vault | Hôm nay/tuần này học gì, cần nối gì? |
