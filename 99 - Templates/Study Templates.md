# Study Templates

Bộ template này dùng cho việc học một lĩnh vực từ sách, course, paper hoặc video dài.

Xem quy trình học đầy đủ tại [[Cách học một domain từ sách]].

## Templates

| Template | Dùng khi nào | Folder thường dùng |
|---|---|---|
| `Domain MOC.md` | Khi học một lĩnh vực / nhánh kiến thức | `02 - Domains` |
| `Book Note.md` | Khi đọc một quyển sách | `05 - Sources/Books` |
| `Concept Note.md` | Khi cần hiểu sâu một khái niệm | `04 - Notes/Concepts` |
| `Atomic Note.md` | Khi có một insight dạng mệnh đề | `04 - Notes/Atomic` |
| `Section Study Note.md` | Khi một section sách quá quan trọng | `05 - Sources/Books` hoặc `00 - Inbox` |
| `Problem Note.md` | Khi giải một bài tập đại diện | `04 - Notes` hoặc `08 - Review` |
| `Error Log Note.md` | Khi muốn gom lỗi theo domain | `08 - Review` |
| `Study Project Note.md` | Khi có kế hoạch học nhiều tuần | `06 - Projects/Study Plans` |
| `Open Questions Note.md` | Khi có câu hỏi chưa giải quyết | `08 - Review` |
| `Prerequisite Gaps Note.md` | Khi thiếu nền | `08 - Review` |

## Luồng dùng nhanh

```text
1. Tạo Book Note
2. Tạo / cập nhật Domain MOC
3. Tạo Study Project nếu sách khó hoặc học dài hạn
4. Mỗi chương extract concept + equation + problem type
5. Tạo Concept Note cho khái niệm lõi
6. Tạo Atomic Note cho insight dạng mệnh đề
7. Làm bài và ghi Problem Note
8. Ghi lỗi vào Error Log
9. Cập nhật Domain MOC mỗi tuần
```

## Nguyên tắc

- Source note giữ nguyên liệu.
- Domain MOC giữ bản đồ.
- Concept note giữ hiểu sâu.
- Atomic note giữ insight nhỏ.
- Problem note kiểm tra khả năng dùng kiến thức.
- Error Log biến sai lầm thành tài sản học tập.
