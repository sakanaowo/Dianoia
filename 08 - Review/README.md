# 08 - Review

Review là nơi kiểm tra, sửa, kết nối và làm sắc hệ thống tri thức.

Nếu `04 - Notes` là trái tim của vault, thì `08 - Review` là nơi giúp vault không bị mục nát.

## Nên có các file

```text
To Process.md
To Connect.md
Open Questions.md
Contradictions.md
Weak Notes.md
Weekly Review.md
```

## To Process

Danh sách note hoặc ý tưởng cần xử lý.

Ví dụ:

```md
- [ ] Tách note từ [[NLP - Buổi 7]]
- [ ] Chuyển ghi chú về Nietzsche thành atomic notes
- [ ] Xử lý lỗi CMake multiple definition thành Problem note
```

## To Connect

Danh sách note cần thêm liên kết.

Ví dụ:

```md
- [[Entropy]] cần nối với [[Temperature làm phân phối token sắc hoặc phẳng hơn]]
- [[Free Will]] cần nối với [[Determinism]]
- [[Automation]] cần nối với [[Technology and Humanity]]
```

## Open Questions

Các câu hỏi chưa trả lời xong.

Ví dụ:

```md
- [[Con người có tự do ý chí không]]
- [[AI có thể hiểu ngôn ngữ hay chỉ dự đoán token]]
- [[Công bằng có cần bình đẳng không]]
```

## Contradictions

Nơi ghi các mâu thuẫn, xung đột hoặc lập luận đối nghịch.

Ví dụ:

```md
## Tự do cá nhân vs trật tự xã hội

- [[Tự do cá nhân là điều kiện cho sáng tạo]]
- [[Xã hội cần giới hạn tự do để duy trì ổn định]]

Câu hỏi:
- Giới hạn nào là hợp lý?
- Ai có quyền đặt ra giới hạn?
```

## Weekly Review

Mỗi tuần nên kiểm tra:

- Inbox còn gì chưa xử lý?
- Note nào đang bị cô lập?
- MOC nào cần cập nhật?
- Có câu hỏi lớn nào mới xuất hiện?
- Có output nào có thể tạo từ các note hiện có?

## Luật cho Review

- Review không phải nơi tự trách bản thân vì chưa hoàn hảo.
- Review dùng để làm hệ thống rõ hơn, nhẹ hơn, sắc hơn.
- Mâu thuẫn không phải lỗi; nó là dấu hiệu của tư duy nghiêm túc.
