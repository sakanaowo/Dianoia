# 06 - Projects

Projects là nơi quản lý những việc có mục tiêu hoàn thành rõ ràng.

Ví dụ:

```text
AI Projects
Writing Projects
Study Plans
Personal Research
Exam Prep
CodePTIT
Dragon Fruit Disease Classification
Action Detection
Chat Application
```

## Project khác Area như thế nào?

Project có điểm kết thúc rõ ràng.

Ví dụ:

```text
Hoàn thành báo cáo ResNet-50
Ôn xong NLP cho kỳ thi
Xây xong Chat Application
Viết xong bài essay về free will
```

Area là lĩnh vực cần duy trì lâu dài.

Ví dụ:

```text
AI
Health
Programming
Philosophy
English
```

## Mỗi project nên có gì?

Một project nên có file overview:

```md
# {{project name}}

## Mục tiêu

## Vì sao làm project này?

## Kết quả mong muốn

## Trạng thái hiện tại

## Việc cần làm

- [ ] 

## Tài nguyên liên quan

- [[...]]

## Note kiến thức liên quan

- [[...]]

## Output cuối cùng

- [[...]]
```

## Cách dùng project note

Project note không phải nơi ghi tất cả kiến thức.

Nó là nơi điều phối:

```text
Project note → link tới source note
Project note → link tới atomic note
Project note → link tới output
Project note → checklist việc cần làm
```

## Luật cho Projects

- Mỗi project nên có mục tiêu rõ.
- Nếu project không còn hoạt động, chuyển sang Archive.
- Nếu có kiến thức quan trọng phát sinh từ project, tách sang `04 - Notes`.
- Nếu có sản phẩm hoàn chỉnh, đưa sang `07 - Outputs`.
