A MOC is a map, not a textbook.

When building a MOC:

1. Keep it navigational.
2. Group links by meaning, not alphabetical order only.
3. Include important questions.
4. Include core concepts.
5. Include opposing ideas or contradictions when relevant.
6. Link to projects and outputs if useful.
7. Avoid dumping every related note into the MOC.
8. Prefer sections such as:
   - Tổng quan
   - Câu hỏi trung tâm
   - Kiến thức nền
   - Chủ đề con
   - Note quan trọng
   - Mâu thuẫn / tranh luận
   - Project liên quan
   - Output liên quan

For theme MOCs, connect multiple domains.
For domain MOCs, show the internal structure of one field.
